## 一、系统概述

### 1.1 项目背景

**团队项目协作管理系统** 是一个面向小型开发团队、课程项目组、兴趣小组的项目管理平台。系统以 **团队组建 → 阶段规划 → 任务分配 → 进度追踪** 为主线，贯穿项目协作的全生命周期，帮助团队高效组织工作、追踪进度、完成目标。

### 1.2 核心业务流程

```
用户注册 ──→ 创建/加入团队 ──→ 创建项目 ──→ 定义阶段
                                                │
                                         任务看板协作
                                                │
                                         阶段进度追踪
                                                │
                                         项目完成归档
```

### 1.3 角色定义

| 角色 | 权限范围 |
|------|----------|
| **系统管理员** | 用户管理、技能库维护、全局团队管理 |
| **队长 (Leader)** | 团队信息编辑、审核入队申请、移除成员、创建项目与阶段、分配任务 |
| **普通成员 (Member)** | 查看团队/项目/任务、认领/执行任务、移动任务状态 |
| **游客** | 注册、浏览招募墙（受限） |

### 模块一：用户与个人中心

#### 1.1 账户管理

| 功能 | 描述 | 核心逻辑 |
|------|------|----------|
| **注册** | 用户填写用户名、密码、邮箱，并可选选择技能标签（多选）。调用 `user_skills` 关联插入。 | 用户名、邮箱唯一校验；密码 bcrypt 哈希存储 |
| **登录/登出** | JWT 鉴权，Token 有效期 7 天；登出时前端清除 Token。 | 登录成功返回 `{ access_token, user_info }` |
| **个人资料编辑** | 修改昵称、头像 URL、个人简介（bio）。 | `UPDATE users SET ... WHERE user_id = ?` |
| **密码修改** | 需验证原密码，新密码长度 ≥ 6 位。 | 原密码比对后更新哈希 |

#### 1.2 技能体系

| 功能 | 描述 | 核心逻辑 |
|------|------|----------|
| **系统技能标签库** | 管理员维护全局技能池（如 Python、React、UI 设计），所有用户和团队共用。 | `skills` 表，管理员可增删 |
| **个人技能管理** | 用户从技能库中选择添加自己的技能（支持自定义熟练度：初学/掌握/精通）。 | `user_skills` 多对多关联，前端级联选择器 |

**技能库预设分类：**

| 分类 | 示例标签 |
|------|----------|
| 前端 | HTML/CSS、JavaScript、React、Vue、TypeScript |
| 后端 | Java、Python、Go、Node.js、Spring Boot |
| 数据库 | MySQL、PostgreSQL、MongoDB、Redis |
| 移动端 | Android、iOS、Flutter、React Native |
| 设计 | UI Design、UX Research、Figma、Photoshop |
| DevOps | Docker、K8s、CI/CD、AWS、Linux |
| 其他 | 项目管理、测试、技术写作、数据分析 |

#### 1.3 个人工作台

| 功能 | 描述 | 核心逻辑 |
|------|------|----------|
| **我的团队列表** | 展示用户所属的所有团队（含角色标签：队长/成员）。 | `SELECT t.* FROM teams t JOIN team_members tm ON t.team_id = tm.team_id WHERE tm.user_id = ?` |
| **我的申请记录** | 展示用户发出过的所有入队申请及其状态（待审核/已通过/已拒绝）。 | `SELECT * FROM join_requests WHERE user_id = ? ORDER BY created_at DESC` |
| **我的任务** | 跨所有团队聚合展示分配给当前用户的任务，按项目/团队分组。 | 多表 JOIN 查询，支持按状态、优先级筛选 |
| **我的通知** | 汇总所有通知，支持已读/未读筛选，红点提示未读数。 | `SELECT * FROM notifications WHERE user_id = ? ORDER BY created_at DESC` |

### 模块二：团队组建与管理

#### 2.1 团队创建

| 功能 | 描述 | 核心逻辑 |
|------|------|----------|
| **创建团队** | 队长填写名称、简介、计划人数（max_members）、所需技能（多选），团队默认状态为"招募中"。 | 事务写入 `teams` + `team_required_skills` |
| **编辑信息** | 队长可修改名称、简介、计划人数、所需技能。 | `UPDATE teams` + 先删后插 `team_required_skills` |
| **解散团队** | 队长操作，级联删除成员关系、项目数据。 | 事务中逐层清理：notifications → tasks → task_lists → stages → projects → team_members → join_requests → team_required_skills → teams |

#### 2.2 团队发现

| 功能 | 描述 | 核心逻辑 |
|------|------|----------|
| **招募墙** | 公开展示所有状态为"招募中"的团队列表卡片（名称、简介、技能标签、当前/计划人数）。 | `SELECT t.*, COUNT(tm.user_id) AS current_members FROM teams t LEFT JOIN team_members tm ... WHERE t.status = 'recruiting' GROUP BY t.team_id` |
| **组合筛选** | 支持按技能标签、人数范围、团队名称关键词多重筛选。 | 动态 SQL 拼接 |
| **团队详情** | 展示完整团队信息：基本信息、所需技能、成员列表（头像、角色）、申请入口。 | 多表聚合查询 |

#### 2.3 成员管理

| 功能 | 描述 | 并发/约束 |
|------|------|-----------|
| **入队申请** | 用户在招募墙或详情页提交申请，可附带留言说明自己的优势。 | **唯一约束**: `UNIQUE(user_id, team_id)` 防止重复申请；检查当前成员数 < max_members |
| **队长审核** | 队长在通知/申请列表中逐条审核，点击通过或拒绝。 | **行级锁**: `SELECT ... FOR UPDATE` 锁定团队行，通过前校验 `COUNT(members) < max_members`，通过后 `INSERT INTO team_members`，事务提交释放锁 |
| **移除成员** | 队长可将非队长成员移出团队。 | `DELETE FROM team_members WHERE team_id = ? AND user_id = ? AND role != 'leader'` |

### 模块三：多阶段项目管理

#### 3.1 阶段定义

| 功能 | 描述 | 核心逻辑 |
|------|------|----------|
| **创建阶段** | 在项目下创建阶段，包含名称、描述、开始日期、截止日期，自动分配 `order_index`（上阶段 +1）。 | `INSERT INTO stages` |
| **编辑/删除阶段** | 修改阶段信息；删除前校验该阶段下无未完成任务（或提供级联选项）。 | 软检查或 `ON DELETE SET NULL` |
| **使用模板** | 系统预置常用模板（瀑布模型：需求→设计→开发→测试→部署），一键填充到当前项目。 | `stage_templates` 表存储 JSON 结构，创建时批量 INSERT |

**预置阶段模板：**

| 模板名称 | 包含阶段 |
|----------|----------|
| 标准瀑布模型 | 需求分析 → 系统设计 → 编码开发 → 测试验收 → 部署上线 |
| 敏捷迭代 | Sprint 1 → Sprint 2 → Sprint 3 → Sprint 回顾 |
| 课程设计 | 开题报告 → 中期检查 → 成果开发 → 答辩准备 |

#### 3.2 阶段追踪

| 功能 | 描述 | 核心逻辑 |
|------|------|----------|
| **阶段状态** | 3 种状态：`not_started`（灰色）、`in_progress`（蓝色）、`completed`（绿色）。手动启动/完成，或根据任务完成率自动计算。 | 状态字段 + 可选自动流转 |
| **阶段进度** | `任务完成率 = 该阶段下已完成任务数 / 该阶段下任务总数 × 100%` | `SELECT COUNT(*) FILTER(WHERE status='done') / COUNT(*) FROM tasks WHERE stage_id = ?` |
| **超期提醒** | 当前日期 > 阶段 `end_date` 且状态 ≠ `completed` 时，该阶段标记为超期（红色），并向队长发送通知。 | 定时任务/查询时动态计算 |

### 模块四：任务看板与协作

#### 4.1 看板管理

| 功能 | 描述 | 核心逻辑 |
|------|------|----------|
| **默认列表** | 项目创建时自动生成三个不可删除的列表：**待办**（position=0）、**进行中**（position=1）、**已完成**（position=2）。 | `is_default = true`，前端隐藏删除按钮 |
| **自定义列表** | 队长可新增自定义列表（如"代码审查中"、"待测试"），支持拖拽排序修改 position。 | `position` 字段 + 批量更新 |

#### 4.2 任务卡片

| 功能 | 描述 | 核心逻辑 |
|------|------|----------|
| **创建任务** | 在指定列表中创建任务卡片：标题（必填）、描述、优先级（高/中/低）、截止日期、执行人、所属阶段（可选）。 | `INSERT INTO tasks` |
| **编辑任务** | 修改任务所有字段。 | `UPDATE tasks` |
| **移动任务** | 拖拽卡片到其他列表（修改 `list_id`）；或在同一列表内调整排序（修改 `position`）。 | 单字段更新或批量重排 |
| **删除任务** | 软删除或硬删除（确认弹窗）。 | `DELETE FROM tasks WHERE task_id = ?` |
| **任务详情** | 弹窗/侧边栏展示完整任务信息与操作日志。 | 单表 + 关联查询 |

**任务优先级颜色标识：**

| 优先级 | 颜色 | 说明 |
|--------|------|------|
| 高 (high) | 🔴 红色 | 紧急，需立即处理 |
| 中 (medium) | 🟡 黄色 | 正常，按计划推进 |
| 低 (low) | 🟢 绿色 | 可延后，有空再处理 |

#### 4.3 分配与追踪

| 功能 | 描述 | 核心逻辑 |
|------|------|----------|
| **指派执行人** | 从团队成员中选择一人作为任务执行人（assignee）。变更时向新旧执行人发送通知。 | `UPDATE tasks SET assignee_id = ?` + 通知生成 |
| **排序** | 看板内任务支持按优先级、截止日期、创建时间排序。 | `ORDER BY FIELD(priority, 'high','medium','low'), due_date ASC` |
| **截止日期通知** | 截止日期前 24 小时，系统向执行人和任务创建者发送提醒通知。 | 定时任务扫描 `tasks WHERE due_date BETWEEN NOW() AND NOW() + INTERVAL 1 DAY` |

#### 4.4 进度统计

| 统计指标 | 计算公式 |
|----------|----------|
| **团队完成率** | `已完成任务数 / 团队所有任务总数 × 100%` |
| **个人完成数** | `COUNT(tasks) WHERE assignee_id = ? AND status = 'done'` |
| **个人超时数** | `COUNT(tasks) WHERE assignee_id = ? AND status != 'done' AND due_date < NOW()` |
| **个人贡献度** | `(个人完成数 / 团队总完成数) × 100%` |

### 模块五：通知与消息中心

#### 5.1 通知类型

| 类型 | 触发场景 | 通知接收人 |
|------|----------|------------|
| **审核结果通知** | 队长通过/拒绝入队申请 | 申请人 |
| **新申请通知** | 用户提交入队申请 | 队长（该团队所有 leader） |
| **任务分配通知** | 被指派为任务执行人 | 被指派人 |
| **任务变更通知** | 任务被移动/修改 | 任务执行人、创建者 |
| **截止日期提醒** | 任务/阶段即将到期 | 执行人、队长 |
| **超期通知** | 阶段已超期 | 队长 |
| **成员移除通知** | 被移出团队 | 被移除人 |

#### 5.2 消息中心

| 功能 | 描述 |
|------|------|
| **通知列表** | 按时间倒序展示，未读通知高亮加粗，已读通知灰色。 |
| **标记已读** | 支持单条标记已读（点击进入详情时）和全部标记已读（一键操作）。 |
| **未读计数** | 顶部导航栏红点数字，WebSocket 实时推送或轮询。 |
| **通知跳转** | 点击通知可跳转到关联的任务/申请/团队详情页（通过 `related_type` + `related_id` 定位）。 |

### 模块六：系统管理与安全

#### 6.1 管理员功能

| 功能 | 描述 |
|------|------|
| **用户管理** | 查看用户列表、禁用/启用账号、重置密码。 |
| **技能库维护** | 新增/删除/编辑系统技能标签。 |
| **团队管理** | 查看所有团队、强制解散违规团队。 |

#### 6.2 安全机制

| 机制 | 实现方式 |
|------|----------|
| **接口鉴权 (JWT)** | 登录后颁发 `access_token`，请求头携带 `Authorization: Bearer <token>`，中间件校验。 |
| **行级锁 (防超募)** | 审核入队申请时使用 `SELECT ... FOR UPDATE` 锁定团队行，确保并发场景下不会超过 `max_members`。 |
| **防重唯一约束** | `user_skills(user_id, skill_id)` 防止重复添加技能；`join_requests(user_id, team_id)` 防止重复申请；`team_members(user_id, team_id)` 防止重复入队。 |
| **密码安全** | bcrypt 加盐哈希，不可逆存储。 |
| **SQL 注入防护** | 全部使用参数化查询（Prepared Statements）。 |

### 3.3 表结构设计

#### 3.3.1 用户表 `users`

```sql
CREATE TABLE users (
    user_id       BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    username      VARCHAR(50)   NOT NULL UNIQUE,
    password_hash VARCHAR(255)  NOT NULL,
    email         VARCHAR(100)  NOT NULL UNIQUE,
    avatar        VARCHAR(500)  DEFAULT NULL      COMMENT '头像URL',
    bio           VARCHAR(500)  DEFAULT NULL      COMMENT '个人简介',
    role          ENUM('admin','member') NOT NULL DEFAULT 'member',
    created_at    DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at    DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_users_email (email),
    INDEX idx_users_username (username)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

#### 3.3.2 技能表 `skills`

```sql
CREATE TABLE skills (
    skill_id    BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
```

#### 触发器 3: 新申请通知队长

```sql
DELIMITER $$
CREATE TRIGGER trg_join_requests_after_insert
AFTER INSERT ON join_requests
FOR EACH ROW
BEGIN
    DECLARE done INT DEFAULT FALSE;
    DECLARE leader_id BIGINT UNSIGNED;
    DECLARE cur CURSOR FOR
        SELECT user_id FROM team_members
        WHERE team_id = NEW.team_id AND role = 'leader';
    DECLARE CONTINUE HANDLER FOR NOT FOUND SET done = TRUE;

    OPEN cur;
    read_loop: LOOP
        FETCH cur INTO leader_id;
        IF done THEN
            LEAVE read_loop;
        END IF;
        INSERT INTO notifications (user_id, type, title, content, related_type, related_id)
        VALUES (
            leader_id,
            'new_join_request',
            '收到新的入队申请',
            CONCAT('用户 ',
                (SELECT username FROM users WHERE user_id = NEW.user_id),
                ' 申请加入团队 "',
                (SELECT name FROM teams WHERE team_id = NEW.team_id),
                '"'),
            'join_request',
            NEW.request_id
        );
    END LOOP;
    CLOSE cur;
END$$
DELIMITER ;
```

### 3.6 并发控制与事务设计

#### 3.6.1 入队审核 —— 防超募

```sql
-- 伪代码：队长审核通过入队申请的事务流程

START TRANSACTION;

-- 步骤 1: 行级锁锁定目标团队
SELECT max_members FROM teams
WHERE team_id = :team_id
FOR UPDATE;

-- 步骤 2: 检查当前成员数
SELECT COUNT(*) AS current_count FROM team_members
WHERE team_id = :team_id;

-- 步骤 3: 若 current_count >= max_members，ROLLBACK 并返回"团队已满"
-- 步骤 4: 更新申请状态为 approved
UPDATE join_requests SET status = 'approved', reviewed_at = NOW()
WHERE request_id = :request_id;

-- 步骤 5: 插入团队成员记录（PRIMARY KEY 唯一约束防止重复）
INSERT INTO team_members (team_id, user_id, role, joined_at)
VALUES (:team_id, :user_id, 'member', NOW());

-- 步骤 6: 若团队满员，自动将团队状态改为 closed
UPDATE teams SET status = 'closed'
WHERE team_id = :team_id AND
      (SELECT COUNT(*) FROM team_members WHERE team_id = :team_id) >= max_members;

COMMIT;
```

## 附录：建表顺序

由于外键依赖，建表顺序必须为：

```
1. users
2. skills
3. user_skills         (→ users, skills)
4. teams               (→ users)
5. team_required_skills (→ teams, skills)
6. team_members        (→ teams, users)
7. join_requests       (→ teams, users)
8. projects            (→ teams)
9. stages              (→ projects)
10. stage_templates    (无外键)
11. task_lists         (→ projects)
12. tasks              (→ task_lists, stages, users)
13. notifications      (→ users)
14. 触发器创建
```
